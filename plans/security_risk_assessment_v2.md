# 專案潛在資安風險評估報告 - V2 最終確認

## 1. 核心風險點狀態更新

| 編號 | 資安風險點 | 說明 | 現狀與風險等級 | 緩解措施 (待實作) |
| :--- | :--- | :--- | :--- | :--- |
| **S1** | XSS (Cross-Site Scripting) | 前端數據輸出未經轉義。 | **低風險 (Low)**。 Vue.js 預設機制和後端 `sanitize` 函式已緩解。 | 無需緊急處理。 |
| **S2** | Formula Injection (公式注入) | 寫入 SpreadSheet 的數據包含惡意公式。 | **低風險 (Low)**。 寫入端 (如 [`src/api_web_gui.js`](src/api_web_gui.js:299)) 已有 `sanitize` 函式進行清洗。 | 無需緊急處理。 |
| **S3** | **Web App 權限洩漏** | 執行身分為 **部署者本人**。 | **極高風險 (CRITICAL)**。 任何知道 Web App URL 的人都可以使用部署者的權限 (讀寫所有數據、發送郵件等)。 | 必須將部署身分改為「存取 Web App 的使用者」並在 `doGet`/`doPost` 中加入嚴格的身份驗證。 |
| **S4** | **Line Bot 白名單繞過** | `ALLOWED_USERS` 硬編碼在程式碼中。 | **中風險 (Medium)**。維護性差且缺乏動態控制。 | 應將白名單移至 Spreadsheet/Cache (已完成規劃)。 |
| **S5** | 配置屬性洩露 | 程式碼將機密配置輸出到前端。 | **低風險 (Low)**。 DAO/Config 層已將配置隔離開。 | 無需緊急處理。 |
| **S6** | **DDOS/資源耗盡** | 昂貴的 API (如 `apiGetGlobalSummary`) 無速率限制。 | **中風險 (Medium)**。惡意呼叫可能耗盡 GAS 每日配額。 | 需實作 API 節流 (Throttling) 或 IP 限制 (複雜)。 |

## 2. 下一步行動建議 (實作計畫)

我們建議優先解決 **S3 (權限洩漏)**，因為這是最嚴重的資安漏洞。一旦修復 S3，就可以安全地繼續實作 S4 和 S6。

請選擇接下來的實作方向：

*   **A.** **Web App 權限升級 (S3)**：將執行身分改為「使用者」並在代碼中強制檢查管理員白名單。
*   **B.** **Line Bot 白名單優化 (S4)**：實作 Spreadsheet + Cache 方案。
*   **C.** **Web GUI 預熱觸發器**：實作性能優化（解決冷啟動問題）。
